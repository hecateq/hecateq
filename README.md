### 🧠 Engineering Principles

*   **Scan First, Change Later** — Deeply understand codebase layout, dependencies, and architecture before making mutations.
*   **No Fake Fallbacks** — Fail explicitly and report immediately instead of implementing silent mock successes.
*   **Test Before Claiming Completion** — Verify changes using project-native runtimes, lint rules, and unit tests.
*   **Deterministic Safety Over Blind Speed** — Prioritize clean, traceable, and secure automation loops over rapid execution.
